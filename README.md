# clus_reports

Network reports generated by Ansible and published from [the_clus_demo](https://github.com/KB-perByte/the_clus_demo).

## View reports

After GitHub Pages is enabled, reports are available at:

**https://kb-perbyte.github.io/clus_reports/**

Each HTML report is also reachable directly, for example:

**https://kb-perbyte.github.io/clus_reports/reports/meraki_org_828099381482762270_report.html**

## GitHub Pages setup (one-time)

1. Open **Settings → Pages** for this repository.
2. Under **Build and deployment**, set **Source** to **GitHub Actions**.
3. Push a commit that updates `reports/` (or run the **Publish reports to GitHub Pages** workflow manually).

The [pages workflow](.github/workflows/pages.yml) runs on every push to `main` that changes files under `reports/`.

## Repository layout

| Path | Description |
|------|-------------|
| `reports/*.html` | Human-readable Meraki reports (served on GitHub Pages) |
| `reports/*.yaml` | Raw report data |
